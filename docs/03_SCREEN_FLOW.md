# BLC Care Screen Flow

## 1. 문서 개요

본 문서는 BLC Care의 화면 흐름과 주요 페이지 구조를 정의한다.

BLC Care는 다음 사용자 흐름을 가진다.

* 로그인 전 사용자
* Admin
* 셀리더
* Admin과 셀리더 역할을 동시에 가진 사용자
* 새신자

각 화면은 PC와 모바일에서 모두 접근 가능해야 한다.
셀리더는 모바일 사용성을 우선 고려한다.

---

# 2. 전체 라우팅 구조

```txt
/
├─ /login
├─ /newcomer
├─ /newcomer/complete
├─ /dashboard
├─ /members
├─ /members/[id]
├─ /reports
├─ /reports/new
├─ /reports/[id]
└─ /admin
   ├─ /admin/dashboard
   ├─ /admin/cells
   ├─ /admin/users
   ├─ /admin/newcomers
   ├─ /admin/absence
   ├─ /admin/settings
   └─ /admin/backup
```

---

# 3. 사용자 role 처리 원칙

## 3.1 role 구조

한 사용자는 하나 이상의 role을 가질 수 있다.

예시:

```txt
admin
cell_leader
admin,cell_leader
```

## 3.2 접근 원칙

| 사용자 role          | 기본 진입 화면           | 접근 범위               |
| ----------------- | ------------------ | ------------------- |
| admin             | `/admin/dashboard` | 전체 데이터              |
| cell_leader       | `/dashboard`       | 배정된 셀 데이터           |
| admin,cell_leader | `/admin/dashboard` | 전체 데이터 + 배정된 셀리더 기능 |

## 3.3 Admin + 셀리더 겸임자 처리

사용자가 `admin`과 `cell_leader` role을 동시에 가진 경우:

* 기본 진입은 `/admin/dashboard`로 한다.
* Admin 메뉴 전체에 접근할 수 있다.
* 셀리더 기능에도 접근할 수 있다.
* `/dashboard`, `/reports/new`에서는 본인에게 배정된 셀을 기본 선택한다.
* 여러 셀에 배정되어 있으면 셀 선택 드롭다운을 표시한다.

# 4. 사용자별 기본 흐름

## 4.1 로그인 전 사용자

```txt
/ 접속
 → 구글 로그인 선택
   → /login
   → 로그인 성공
   → 사용자 role 확인
   → admin 포함 시 /admin/dashboard
   → cell_leader만 있으면 /dashboard

/ 접속
 → 새신자 등록 선택
   → /newcomer
   → 등록폼 제출
   → /newcomer/complete
```

---

## 4.2 Admin 흐름

```txt
구글 로그인
 → roles에 admin 포함 확인
 → /admin/dashboard
```

Admin 주요 흐름:

```txt
성도 관리:
 /admin/dashboard
 → /members
 → /members/[id]
 → 성도 정보 수정 / 휴면 처리 / 특이사항 확인

새신자 관리:
 /admin/dashboard
 → /admin/newcomers
 → 새신자 상세 확인
 → 성도 DB 전환

장기결석자 관리:
 /admin/dashboard
 → /admin/absence
 → 장기결석자 확인
 → 조치 메모 작성
 → 확인 또는 해결 처리

셀 관리:
 /admin/dashboard
 → /admin/cells
 → 셀 생성/수정/비활성화

사용자 및 셀 배정 관리:
 /admin/dashboard
 → /admin/users
 → 사용자 등록
 → role 지정
 → 담당 셀 배정
```

---

## 4.3 셀리더 흐름

```txt
구글 로그인
 → roles에 cell_leader 포함 확인
 → roles에 admin 없음
 → /dashboard
```

셀리더 주요 흐름:

```txt
이번 주 셀 리포트 작성:
 /dashboard
 → /reports/new
 → 담당 셀 선택 또는 자동 선택
 → 셀 인원 자동 로드
 → 출결 입력
 → 나눔 요약 입력
 → 기도제목 일괄 입력
 → 자동 분리
 → 결과 확인
 → 저장 또는 제출

성도 히스토리 확인:
 /dashboard
 → /members
 → 본인에게 배정된 셀 인원만 표시
 → /members/[id]
 → 출결/나눔/기도제목/특이사항 확인

특이사항 기록:
 /members/[id]
 → 특이사항 추가
 → 해결 여부 체크
```

---

## 4.4 Admin + 셀리더 겸임자 흐름

```txt
구글 로그인
 → roles에 admin, cell_leader 둘 다 포함
 → /admin/dashboard
```

셀리더 기능 접근:

```txt
/admin/dashboard
 → 상단 메뉴 또는 사이드 메뉴의 "내 셀 리포트"
 → /dashboard 또는 /reports/new
 → 본인 배정 셀 선택
 → 리포트 작성
```

겸임자 규칙:

* Admin 화면에서는 전체 데이터 기준으로 본다.
* 셀리더 화면에서는 본인에게 배정된 셀을 기본값으로 본다.
* 단, Admin 권한이 있으므로 필요 시 전체 셀 선택도 가능하게 할 수 있다.
* UI에서는 현재 화면이 Admin 모드인지 셀리더 모드인지 구분되어야 한다.

예시 표시:

```txt
현재 모드: Admin
현재 모드: 셀리더
```

---

## 4.5 새신자 흐름

```txt
/ 접속
 → 새신자 등록 버튼
 → /newcomer
 → 등록폼 작성
 → 개인정보 수집 동의 체크
 → 제출
 → /newcomer/complete
```

새신자는 로그인하지 않는다.

# 5.2 `/login`

## 목적

구글 로그인을 처리한다.

## 동작

1. 구글 로그인 실행
2. 로그인 이메일 확인
3. `users` 데이터에서 계정 조회
4. `roles` 확인
5. role에 따라 이동

## 이동 규칙

| roles             | 이동                 |
| ----------------- | ------------------ |
| admin             | `/admin/dashboard` |
| cell_leader       | `/dashboard`       |
| admin,cell_leader | `/admin/dashboard` |
| 미등록 사용자           | 접근 불가 안내           |

## 미등록 사용자 안내

```txt
등록되지 않은 계정입니다.
관리자에게 문의해주세요.
```

# 5.3 `/newcomer`

## 목적

새신자가 로그인 없이 정보를 제출하는 화면이다.

## 입력 항목

* 이름
* 휴대폰 번호
* 주소
* 방문동기
* 방문경로
* 신앙생활 경험
* 예배 이후 일정
* 개인정보 수집 동의

## 필수 항목

* 이름
* 휴대폰 번호
* 개인정보 수집 동의

## 버튼

| 버튼 | 동작                                        |
| -- | ----------------------------------------- |
| 제출 | `newcomers`에 저장 후 `/newcomer/complete` 이동 |

## 유효성 체크

* 이름은 비어 있으면 안 된다.
* 휴대폰 번호는 비어 있으면 안 된다.
* 개인정보 수집 동의가 체크되어야 한다.

```txt
등록이 완료되었습니다.
방문해주셔서 감사합니다.
```

## 버튼

| 버튼   | 이동  |
| ---- | --- |
| 처음으로 | `/` |

# 5.5 `/dashboard`

## 목적

셀리더용 대시보드다.

Admin이면서 셀리더인 사용자는 이 화면에 접근할 수 있다.
Admin role만 있고 셀리더 role이 없는 사용자가 접근하면 `/admin/dashboard`로 이동시킬 수 있다.

## 표시 요소

* 현재 모드: 셀리더
* 담당 셀 선택
* 이번 주 리포트 작성 상태
* 담당 셀 인원 수
* 최근 리포트
* 미해결 특이사항
* 장기결석 의심 인원

## 담당 셀 선택 규칙

| 배정 셀 수 | 동작              |
| ------ | --------------- |
| 0개     | 담당 셀이 없다는 안내 표시 |
| 1개     | 자동 선택           |
| 2개 이상  | 셀 선택 드롭다운 표시    |

## 주요 버튼

| 버튼          | 이동                 |
| ----------- | ------------------ |
| 이번 주 리포트 작성 | `/reports/new`     |
| 성도 목록 보기    | `/members`         |
| 리포트 목록 보기   | `/reports`         |
| Admin 대시보드  | `/admin/dashboard` |

`Admin 대시보드` 버튼은 admin role이 있는 사용자에게만 표시한다.

* 검색창

* 상태 필터

* 성도 카드 또는 테이블
* 페이지네이션

## 목록 항목

* 사진
* 이름
* 현재 셀
* 상태
* 마지막 출석일
* 미해결 특이사항 여부

## 권한별 표시

| 사용자               | 표시 범위                           |
| ----------------- | ------------------------------- |
| admin             | 전체 성도                           |
| cell_leader       | 배정된 셀 성도                        |
| admin,cell_leader | 기본 전체 성도. 셀리더 모드에서는 배정된 셀 우선 표시 |

## 클릭 동작

성도 항목 클릭 시 `/members/[id]`로 이동한다.

# 5.9 `/reports/new`

## 목적

주차별 셀 리포트 작성 화면이다.

## 기본 동작

1. 로그인 사용자 확인
2. 사용자 role 확인
3. 작성 가능한 셀 목록 확인
4. 현재 주차 계산
5. 선택된 셀 인원 자동 로드
6. 이미 작성된 리포트가 있으면 기존 리포트 불러오기
7. 없으면 새 리포트 작성
8. 출결 한눈에 입력 단계 표시
9. 출결 입력 후 나눔 작성 단계로 이동

## 작성 가능한 셀 기준

| 사용자               | 작성 가능 셀                 |
| ----------------- | ----------------------- |
| admin             | 전체 셀                    |
| cell_leader       | 배정된 셀                   |
| admin,cell_leader | 전체 셀 또는 배정 셀. 기본값은 배정 셀 |

## 셀 선택 규칙

| 상황            | 동작           |
| ------------- | ------------ |
| 작성 가능 셀 0개    | 작성 불가 안내     |
| 작성 가능 셀 1개    | 자동 선택        |
| 작성 가능 셀 2개 이상 | 셀 선택 드롭다운 표시 |

## 입력 영역

### 1. 리포트 기본 정보

* 주차
* 모임일
* 셀 이름
* 작성자

### 2. 출결 한눈에 입력

* 전체 인원의 출결 상태를 한 화면에서 빠르게 선택
* 출석, 결석, 사유 결석, 미확인 인원 수 요약
* 모두 출석 일괄 선택
* 출결 상태 입력 모드를 고른 뒤 명단 그리드에서 해당 인원을 연속 선택
* 기본 입력 모드에서 출석한 사람만 선택하고 미선택 인원은 명시적인 버튼으로 결석 처리
* 미선택 인원을 자동으로 결석 처리하지 않음
* 결석, 사유 결석, 미확인도 입력 모드를 바꾼 뒤 명단에서 선택
* 출결 입력 후 `나눔 작성으로` 버튼으로 다음 단계 이동

### 3. 전체 나눔 요약

* 전체 셀 모임 요약 입력
* 출결 입력 결과 요약과 출결 수정 버튼 표시

### 4. 나눔 및 기도제목 일괄 입력

* 여러 사람의 나눔과 기도제목을 각각 한 번에 붙여넣는 영역
* 자동 분리 버튼
* 분리 결과 확인 영역

### 5. 인원별 기록

각 성도별로 다음 항목을 입력한다.

* 출결 상태 요약 badge
* 개인 나눔 요약
* 기도제목
* 지원 제안

## 버튼

| 버튼         | 동작              |
| ---------- | --------------- |
| 임시저장       | draft 저장        |
| 나눔 작성으로    | 출결 입력 후 나눔 작성 단계로 이동 |
| 출결 수정       | 나눔 작성 단계에서 출결 입력 단계로 이동 |
| 미선택 인원 결석 처리 | 현재 미확인인 인원을 결석으로 변경 |
| 제출         | submitted 저장    |
| 기도제목 자동 분리 | 입력 텍스트를 사람별로 분리 |
| 취소         | 이전 화면 이동        |

기존 리포트의 상세 수정 화면 `/reports/[id]`는 인원별 기록에서 출결과 나눔을 함께 수정하는 기존 흐름을 유지한다.

# 5.10 기도제목 자동 분리 흐름

## 입력 예시

```txt
지현 : 회사 일이 잘 마무리되도록
김진수 : 가족 건강을 위해
주원자매 : 시험 준비를 위해
```

## 처리 흐름

```txt
일괄 입력
 → 자동 분리 버튼 클릭
 → 줄 단위 파싱
 → 이름/기도제목 분리
 → 호칭 제거
 → 선택된 셀 인원 기준 이름 매칭
 → 결과 표시
 → 셀리더 확인
 → 인원별 기도제목 입력칸에 반영
```

## 매칭 결과 상태

| 상태        | 설명              |
| --------- | --------------- |
| matched   | 자동 매칭           |
| ambiguous | 동명이인 또는 후보 여러 명 |
| unmatched | 매칭 실패           |

## 동명이인 처리

```txt
'지현'에 해당하는 인원이 2명 있습니다.
저장할 사람을 선택해주세요.

[장지현] [김지현]
```

## 미매칭 처리

```txt
'민수'를 선택된 셀 인원에서 찾을 수 없습니다.
직접 선택하거나 제외해주세요.
```

## 저장 원칙

자동 분리 결과는 바로 DB에 저장하지 않는다.
사용자가 확인 후 리포트 저장 시 함께 저장한다.

# 5.13 `/admin/members/import`

## 목적

Admin이 CSV 파일로 성도를 Supabase에 일괄 등록하는 화면이다.

## 흐름

```txt
CSV 템플릿 다운로드
→ 최대 500명 작성
→ 파일 선택
→ 브라우저 사전검증
→ 이름·셀·상태 최소 미리보기
→ 사용자 확인 체크
→ Admin 세션 서버 재검증
→ service-role 전용 RPC로 전체 트랜잭션 저장
```

오류가 있으면 행 번호와 원인을 표시하고 아무 행도 저장하지 않는다. 연락처, 주소, 직장,

## 접근 조건

## 표시 요소

* 현재 모드: Admin
* 전체 성도 수
* 활동 성도 수
* 휴면 성도 수
* 셀별 인원 수
* 이번 주 리포트 제출 현황
* 장기결석자 수
* 신규 새신자 수
* 미해결 특이사항 수

## 주요 이동

| 항목        | 이동                             |
| --------- | ------------------------------ |
| 성도 수      | `/members`                     |
| 리포트 제출 현황 | `/reports`                     |

# 7. 모바일 UI 원칙

셀리더는 모바일에서 사용할 가능성이 높다.

따라서 다음 원칙을 적용한다.

* 리포트 작성 화면은 세로 스크롤 중심으로 구성한다.
* 성도별 입력 영역은 카드 형태로 제공한다.
* 버튼은 엄지손가락으로 누르기 쉽게 크게 만든다.
* 기도제목 일괄 입력 영역은 모바일에서도 충분히 크게 제공한다.
* 테이블은 모바일에서 카드형 목록으로 전환한다.
* Admin 화면은 PC 우선이지만 모바일에서도 조회 가능해야 한다.
* 여러 셀 담당자는 모바일에서도 셀 선택이 쉽게 보여야 한다.

# 11. Codex 구현 지시 참고

Codex는 먼저 실제 API 연동 없이 mock data 기반으로 화면 흐름을 구현한다.

초기 구현 범위:

```txt
1. 라우팅 구조 생성
2. 공통 레이아웃 생성
3. roles 기반 화면 분기 mock 구현
4. admin + cell_leader 겸임자 흐름 구현
5. 담당 셀 선택 UI 구현
6. 성도 목록 화면 mock 구현
7. 성도 상세 화면 mock 구현
8. 리포트 작성 화면 mock 구현
9. 나눔 및 기도제목 자동 분리 UI mock 구현
10. 새신자 등록 화면 mock 구현
11. Admin 대시보드 mock 구현
```

실제 Google Sheets 연동은 API 명세 작성 후 진행한다.
