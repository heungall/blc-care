# BLC Care Privacy Policy

## 1. 문서 개요

본 문서는 BLC Care에서 다루는 개인정보와 민감 정보의 수집, 저장, 접근, 보관, 백업, 삭제 원칙을 정의한다.

BLC Care는 교회 출결, 성도 돌봄, 기도제목, 새신자 정보를 관리하는 시스템이므로 개인정보 보호를 최우선으로 고려한다.

---

# 2. 개인정보 처리 원칙

BLC Care는 다음 원칙을 따른다.

```txt
1. 필요한 정보만 수집한다.
2. 허가된 사용자만 접근한다.
3. 셀리더는 배정된 셀 정보만 접근한다.
4. Admin은 전체 정보에 접근할 수 있으나, 운영 목적 내에서만 사용한다.
5. 기도제목과 특이사항은 민감 정보로 취급한다.
6. 외부 AI API에는 개인정보와 기도제목을 전송하지 않는다.
7. 데이터 변경 이력은 audit_logs에 기록한다.
8. Supabase PostgreSQL과 private Storage는 최소 권한과 RLS 정책으로 관리한다.
```

# 5. 개인정보 수집 동의

## 5.1 새신자 등록폼

새신자 등록폼에는 개인정보 수집 동의 체크박스를 둔다.

동의하지 않으면 제출할 수 없다.

## 5.2 동의 문구 예시

```txt
BLC Care는 새신자 안내 및 교회 공동체 돌봄을 위해 이름, 연락처, 주소, 방문동기, 방문경로, 신앙생활 경험, 예배 이후 일정 정보를 수집합니다.

개인정보 수집 및 이용에 동의합니다.
```

# 6. 접근 권한

## 6.1 비로그인 사용자

비로그인 사용자는 다음 화면만 접근할 수 있다.

```txt
/
 /login
 /newcomer
 /newcomer/complete
```

## 6.2 새신자

새신자는 본인이 등록폼에 입력한 내용을 제출할 수 있다.
제출 후에는 시스템 내부 데이터에 접근할 수 없다.

## 6.3 셀리더

셀리더는 본인에게 배정된 셀 정보만 접근할 수 있다.

접근 가능 정보:

* 배정된 셀의 성도 목록
* 배정된 셀의 성도 상세
* 배정된 셀의 출결 히스토리
* 배정된 셀의 나눔 요약
* 배정된 셀의 기도제목
* 배정된 셀의 특이사항

## 6.4 Admin

Admin은 전체 데이터에 접근할 수 있다.

단, Admin은 교회 운영 및 성도 돌봄 목적 내에서만 정보를 조회하고 수정해야 한다.

# 17. Phase 3-5 감사 로그 최소화

Users, Cells, Members, Newcomers 관리 API는 주요 변경을 `audit_logs`에 기록한다.

감사 로그에는 action, target ID, 변경자 ID, 변경 시각과 상태·역할·관련 ID처럼 변경 추적에 필요한 최소 정보만 기록한다. 이름, 이메일, 전화번호, 주소, 방문 동기, 신앙 경험, 관리자 메모 등 개인정보와 민감정보 원문을 `before_value`, `after_value`, `memo`에 복제하지 않는다.

성도 정보 수정 감사 로그에는 변경된 필드 이름과 소속 셀·상태 ID 변화만 기록하며 인적사항 원문은 복제하지 않는다.

새신자 공개 제출 응답은 생성된 ID, 상태, 제출 시각만 반환한다.

---

# 18. Phase 5 로그인 세션과 비밀값

Google OAuth client secret과 `SUPABASE_SECRET_KEY`는 Supabase 또는 서버 환경변수에만 저장하며 저장소와 브라우저에 노출하지 않는다.

Supabase Auth 로그인 세션은 HttpOnly cookie로 관리한다. 앱은 Google access token과 refresh token을 직접 저장하거나 클라이언트 API payload에 전달하지 않는다.

보호 API 요청의 이메일은 브라우저 입력값이 아니라 서버 세션에서 가져온다.

---

# 19. Supabase RLS 및 공개 제출

모든 앱 테이블은 RLS를 활성화하며 비활성 사용자와 비인가 셀리더의 접근을 차단한다.
브라우저의 `anon` 역할에는 앱 테이블 직접 읽기·쓰기 권한을 부여하지 않는다.

공개 새신자 제출은 Next.js 서버 Route Handler에서 필수값, 개인정보 동의, 요청 크기와 rate
limit을 검증한 뒤 서버 전용 secret key로 저장한다. 응답에는 생성 ID, 상태, 제출 시각 외의
입력 개인정보를 반환하지 않는다.

---

# 20. 성도 CSV 일괄 등록

성도 CSV 파일은 등록 요청 처리 목적으로만 브라우저와 Next.js 서버 메모리에서 사용하며 Storage,
저장소, 서버 로그에 보관하지 않는다. 등록 전 미리보기에는 이름, 셀, 상태, 별칭 수만 표시하고
연락처, 주소, 직장, 가족정보, 메모는 표시하지 않는다.

서버는 CSV 전체를 검증한 뒤 하나의 DB 트랜잭션으로 저장한다. 오류 메시지에는 행 번호와 필드
규칙만 포함하며 입력된 개인정보 원문을 포함하지 않는다.
