# BLC Care UI/UX 개선 TODO

한 번에 여러 항목을 섞지 않고 아래 우선순위대로 한 항목씩 구현한다.

## 실행 규칙

- 한 작업은 하나의 UI 번호를 기준으로 구현하고 검증한 뒤 완료 처리한다.
- 각 작업에서 관련 요구사항, 화면 흐름, 권한, 개인정보, 디자인 문서를 함께 갱신한다.
- 공통 상태값, 문구, 스타일은 화면마다 중복 구현하지 않고 공용 헬퍼 또는 컴포넌트로 정리한다.
- 모바일과 데스크톱을 모두 직접 확인한다.
- 작업 성격에 따라 `npm.cmd run lint`, `npm.cmd run typecheck`, `npm.cmd run test`, `npm.cmd run build`, `git diff --check`를 실행한다.
- 완료 기준을 통과하기 전에는 다음 UI 번호를 시작하지 않는다.

- [ ] 공용 카드 변형과 화면별 밀도 기준 정리

문제:

- 대부분의 화면이 유사한 큰 흰색 카드로 구성되어 중요도와 정보 밀도 차이가 약하다.

구현 범위:

- `components/ui.tsx`의 Card에 필요한 최소한의 변형 또는 밀도 규칙을 추가한다.
- 요약, 목록, 입력, 민감 정보 카드의 시각적 역할을 구분한다.
- 단일 블루+슬레이트 기반은 유지하되 상태색과 여백으로 위계를 강화한다.

완료 기준:

- 반복되는 화면별 일회성 카드 스타일이 줄어든다.
- 모바일에서는 불필요하게 큰 카드 여백이 줄고, 주요 행동은 명확하게 보인다.
- 변경된 규칙을 `docs/09_DESIGN_SYSTEM.md`에 기록한다.
